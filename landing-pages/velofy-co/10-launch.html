Features

Small file. Complete framework.

Everything below ships in one 16KB script. Nothing to assemble, nothing to configure.

Fine-grained signals

Updates flow from a signal straight to the DOM nodes that read it. No virtual DOM, no diffing pass.

CSP-safe expressions

Summit interprets expressions itself, never calling eval or new Function, so it runs under strict Content-Security-Policy.

Keyed list rendering

Lists track items by key, so a reorder moves existing nodes instead of rebuilding the whole list.

Cached computed getters

Derived values compute once, cache the result, and recompute only when a dependency actually changes.

Bundled UI library

An accessible component library ships in the same script. Use it as-is, or copy the markup and own it.

Machine-readable docs

llms.txt indexes every docs page for agents, llms-full.txt packs the corpus into one markdown file, and AGENTS.md drops into your repo.

How it works

Add a tag. Write an attribute. Ship.

The whole install is one request. The whole workflow is your editor and a browser.

1

Drop in one script tag

One request, 16KB gzipped, deferred. This line is the entire install.

index.html
<script src="https://cdn.jsdelivr.net/npm/summitjs/dist/summit.min.js" defer></script>
2

Write behavior in your HTML

State, events, and bindings live as attributes on the markup they control.

index.html
<div s-data="{ count: 0 }">
  <button @click="count++">Add one</button>
  <span s-text="count"></span>
</div>
3

Ship it

No build, no bundler, no config. The file you wrote is the file you deploy, and it runs under the strictest CSP you can set.

response header
Content-Security-Policy: script-src 'self'

FAQ

Common questions

Does it work with my framework?

Yes. Summit reads attributes from your markup and touches nothing else, so it drops into static pages, server templates, CMS themes, and pages that never needed React in the first place.

What about CSP?

Summit never calls eval or new Function. Expressions run through its own interpreter, so it works under strict Content-Security-Policy headers, including script-src 'self'.

How big is it?

About 16KB gzipped, with the signal engine, expression interpreter, keyed list rendering, and the bundled UI library included. Zero dependencies.

Why is it good for AI agents?

Behavior lives in the HTML an agent is already writing, with no build pipeline to break. The docs ship machine-readable: llms.txt indexes every page, llms-full.txt packs the corpus into one markdown file, and AGENTS.md drops straight into your repo.
